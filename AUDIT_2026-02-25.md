## Research: Video review + mass screenshot extraction for investigative cameras

> Live web search from this environment is currently unavailable (Brave API key missing), so this section is based on proven industry tooling and workflows.

### Recommended evaluation pipeline
1. **Ingest + normalize**
   - Convert all incoming streams to a standard codec/container (H.264/H.265 MP4), normalized FPS/timebase.

2. **Scene/motion indexing (first pass)**
   - Scene-cut detection + motion scoring to mark high-value segments.
   - Keep low-value static intervals collapsed.

3. **Object/person/vehicle detection (second pass)**
   - Run CV model over candidate segments.
   - Tag frames with object classes, confidence, and track IDs.

4. **Searchable timeline output**
   - Store detections and timestamps in DB.
   - Enable queries like: "red truck between 1:00–3:00 AM".

5. **Batch screenshot extraction**
   - Keyframe snapshots + interval snapshots + event-triggered snapshots.
   - Output naming convention tied to camera/date/time for evidence chain.

### Practical tooling stack
- **ffmpeg**: fast decode/transcode and frame extraction at scale.
- **PySceneDetect**: robust scene boundary detection for segmentation.
- **OpenCV**: motion detection/tracking and preprocessing.
- **Optional ML (YOLO/Detectron)**: object detection and re-ID enhancements.

### Example extraction patterns
- Every N seconds (triage):
  - `ffmpeg -i input.mp4 -vf fps=1/5 shots/frame_%06d.jpg`
- Scene-change based:
  - `ffmpeg -i input.mp4 -vf "select='gt(scene,0.35)'" -vsync vfr scenes/shot_%06d.jpg`
- Contact sheet for fast review:
  - `ffmpeg -i input.mp4 -vf "fps=1/60,scale=320:-1,tile=5x5" contact.jpg`

### Scale architecture suggestion
1. Queue jobs per camera/day segment.
2. Worker pool processes extraction + detections.
3. Store metadata in DB; media in object storage.
4. Analyst UI supports scrub + filter + export evidence pack.
5. Keep immutable audit trail for all generated derivatives.

### Compliance/evidence notes
- Preserve original raw footage untouched.
- Store hashes for original and derived artifacts.
- Log operator, tool version, and transform parameters for legal defensibility.
